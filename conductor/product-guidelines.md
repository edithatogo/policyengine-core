# Product Guidelines

## Public interfaces

- Prefer explicit, deterministic interfaces and version externally consumed
  formats.
- Preserve calculation semantics unless a specification explicitly changes
  them.
- Document portability limits and reject ambiguous input rather than silently
  translating it.

## Documentation

- Write concise technical guidance grounded in current code and tests.
- Distinguish repository facts, hosted-check results, and external claims.
- Describe fork-local work as controlled-fork evidence, not independent
  evidence or upstream acceptance.

## Accessibility and presentation

- Use descriptive headings, links, and alternative text.
- Keep examples readable, copyable, and free of machine-specific paths or
  credentials.
